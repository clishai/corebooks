First-time install note corebooks is not yet code-signed. Your OS will show a security warning the first time you open it.

macOS: Right-click the .dmg → Open → click Open in the dialog that appears.
Windows: Click More info in the SmartScreen prompt → Run anyway.
Linux: Mark the AppImage executable: chmod +x corebooks-linux-x64.AppImage

These warnings will disappear once we add code signing for the v1.0 public release. You can always verify the installer by building from source below.

Double-entry bookkeeping

Chart of accounts, journal entries, trial balance, balance sheet, and income statement — built on a pure accounting engine.

Privacy first

Everything lives on your machine. No analytics, no telemetry, no cloud account required — ever.

Encrypted backups

Export your data encrypted with AES-256-GCM. Your passphrase, your backup. No recovery if you lose the passphrase.

Multi-user ready

Connect a PostgreSQL server for teams that need shared access across multiple devices simultaneously.

Draft entries

Work in progress is saved automatically. Entries are only posted when debits equal credits exactly — no exceptions.

Open source (AGPL-3.0)

Read the code, audit it, fork it, or contribute. Modifications must remain open source.

Build from source

Requires Node.js 20+. No other software needed — SQLite is bundled. 

# clone and install
git clone https://github.com/clishai/corebooks.git
cd corebooks
npm install

# start the app
npm run dev

Open http://localhost:5173 in your browser. Both the API server and the UI start together.
